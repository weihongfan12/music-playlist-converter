# 音乐平台歌单转换工具开发文档

## 1. 项目概述

### 1.1 项目背景
随着用户对音乐平台需求的多样化，许多用户希望在不同音乐平台之间迁移自己的歌单。本项目旨在提供一个便捷的工具，将国内主流音乐平台（网易云音乐、QQ音乐、酷狗音乐等）的歌单转换为国外主流音乐平台（Spotify、Apple Music、YouTube Music等）支持的格式。

### 1.2 项目目标
- 支持主流国内音乐平台歌单解析
- 支持主流国外音乐平台歌单创建
- 提供准确的歌曲匹配算法
- 提供友好的用户界面
- 支持批量转换和导出

### 1.3 目标用户
- 需要迁移歌单的音乐爱好者
- 海外华人用户
- 音乐平台切换用户

## 7. 核心算法

### 7.1 歌曲匹配算法

#### 7.1.1 多维度匹配策略

```python
def calculate_match_score(source_song, target_song):
    """
    计算歌曲匹配得分
    """
    scores = {}
    
    # 1. 歌名匹配 (权重: 40%)
    name_similarity = calculate_string_similarity(
        source_song['name'], 
        target_song['name']
    )
    scores['name'] = name_similarity * 0.4
    
    # 2. 歌手匹配 (权重: 35%)
    artist_similarity = calculate_artist_similarity(
        source_song['artist'],
        target_song['artist']
    )
    scores['artist'] = artist_similarity * 0.35
    
    # 3. 专辑匹配 (权重: 15%)
    album_similarity = calculate_string_similarity(
        source_song.get('album', ''),
        target_song.get('album', '')
    )
    scores['album'] = album_similarity * 0.15
    
    # 4. 时长匹配 (权重: 10%)
    duration_similarity = calculate_duration_similarity(
        source_song.get('duration', 0),
        target_song.get('duration', 0)
    )
    scores['duration'] = duration_similarity * 0.1
    
    # 总分
    total_score = sum(scores.values())
    
    return {
        'total_score': total_score,
        'details': scores
    }
```

#### 7.1.2 字符串相似度算法

```python
from difflib import SequenceMatcher
import re

def normalize_string(text):
    """
    标准化字符串
    """
    # 转小写
    text = text.lower()
    # 移除特殊字符
    text = re.sub(r'[^\w\s]', '', text)
    # 移除多余空格
    text = ' '.join(text.split())
    return text

def calculate_string_similarity(str1, str2):
    """
    计算字符串相似度 (使用Levenshtein距离)
    """
    str1 = normalize_string(str1)
    str2 = normalize_string(str2)
    
    return SequenceMatcher(None, str1, str2).ratio()
```

#### 7.1.3 歌手匹配算法

```python
def calculate_artist_similarity(source_artists, target_artists):
    """
    计算歌手匹配度
    """
    if not source_artists or not target_artists:
        return 0.0
    
    # 标准化歌手名
    source_set = set(normalize_string(a) for a in source_artists)
    target_set = set(normalize_string(a) for a in target_artists)
    
    # 计算交集
    intersection = source_set & target_set
    union = source_set | target_set
    
    # Jaccard相似度
    jaccard = len(intersection) / len(union) if union else 0
    
    # 如果有完全匹配的歌手，提高分数
    if intersection:
        return min(1.0, jaccard + 0.3)
    
    return jaccard
```

class MatchCache {
  private redis: RedisClient;
  
  async getMatchResult(songId: string, platform: Platform) {
    const key = `song:match:${songId}:${platform}`;
    const cached = await this.redis.get(key);
    
    if (cached) {
      return JSON.parse(cached);
    }
    
    return null;
  }
  
  async setMatchResult(songId: string, platform: Platform, result: MatchResult) {
    const key = `song:match:${songId}:${platform}`;
    await this.redis.setex(key, 86400, JSON.stringify(result)); // 24小时过期
  }
}

#### 10.1.1 Dockerfile示例

```dockerfile
# 前端
FROM node:18-alpine AS frontend-builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=frontend-builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf
EXPOSE 80

# 后端
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

#### 10.1.2 docker-compose.yml

```yaml
version: '3.8'

services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
  
  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=mysql://user:pass@db:3306/music_converter
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis
  
  db:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=rootpass
      - MYSQL_DATABASE=music_converter
    volumes:
      - db_data:/var/lib/mysql
  
  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data

volumes:
  db_data:
  redis_data:
```

### 10.2 CI/CD流程

#### 10.2.1 GitHub Actions

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test
      
      - name: Build
        run: npm run build
      
      - name: Deploy to server
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.HOST }}
          username: ${{ secrets.USERNAME }}
          key: ${{ secrets.SSH_KEY }}
          script: |
            cd /app/music-converter
            docker-compose pull
            docker-compose up -d
```

## 11. 测试方案

### 11.1 单元测试

```typescript
import { describe, it, expect } from 'vitest';
import { calculateStringSimilarity } from '@/utils/matcher';

describe('String Similarity', () => {
  it('should return 1 for identical strings', () => {
    const result = calculateStringSimilarity('hello', 'hello');
    expect(result).toBe(1);
  });
  
  it('should return high value for similar strings', () => {
    const result = calculateStringSimilarity('Hello World', 'hello world');
    expect(result).toBeGreaterThan(0.8);
  });
  
  it('should return low value for different strings', () => {
    const result = calculateStringSimilarity('hello', 'world');
    expect(result).toBeLessThan(0.3);
  });
});
```

### 11.2 集成测试

```typescript
import request from 'supertest';
import app from '@/app';

describe('Playlist API', () => {
  it('should parse playlist successfully', async () => {
    const response = await request(app)
      .post('/api/v1/playlist/parse')
      .send({
        url: 'https://music.163.com/playlist?id=12345678',
        platform: 'netease'
      });
    
    expect(response.status).toBe(200);
    expect(response.body.data).toHaveProperty('playlistId');
    expect(response.body.data.songs).toBeInstanceOf(Array);
  });
});
```

## 13. 风险评估

### 13.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 第三方API变更 | 高 | 中 | 封装API调用层、及时跟进更新 |
| 匹配准确率低 | 高 | 中 | 优化算法、引入人工校验 |
| 并发性能问题 | 中 | 中 | 异步处理、限流、缓存 |
| 数据安全问题 | 高 | 低 | 加密存储、安全审计 |

### 13.2 业务风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 平台封禁API | 高 | 中 | 使用官方API、多平台备份 |
| 版权问题 | 中 | 低 | 仅做歌单转换、不涉及音频文件 |
| 用户隐私泄露 | 高 | 低 | 数据加密、隐私政策 |
